claude-token-queue

● 백그라운드 자동 감지 · 무인 재실행
자동 감지
데몬이 세션 기록을 보고 한도(429)를 스스로 감지 — 래퍼 불필요
세션 resume

사용자가 아무것도 안 해도 리셋 시각에 자동 실행

무엇이 언제 실행될지 언제든 확인

어떻게 동작하나 (무인 자동)

  1. 클로드 앱으로 작업하다 토큰이 소진된다 (하나/여러 세션 중단 또는 신규 요청 실패).
  2. 워처 데몬이 트랜스크립트에서 한도 이벤트(isApiErrorMessage+429)를 감지.
  3. 실패한 프롬프트 + cwd + sessionId + 리셋시각(resets 7:40pm (Asia/Seoul))을 추출해 큐에 등록.
  4. 가장 이른 리셋 시각에 launchd 트리거를 건다.
  5. 리셋 시각이 되면 claude --resume <sessionId> -p 로 원래 세션을 이어 자동 재실행.
  6. 언제든 get_plan 으로 "무엇이 언제 실행될지" 확인.

감지는 데몬 설치 이후 발생한 한도만 대상(과거 일괄 실행 방지). 중복은 (sessionId, promptId)로 제거.

설치 — MCP + 감지 데몬 (권장)

1. 고정 설치 (데몬용)

uv tool install --from "git+https://github.com/naryeo628/claude-token-queue.git#subdirectory=mcp-server" claude-token-queue-mcp

2. Claude Code에 MCP 등록

claude mcp add ctq -- ctq-mcp

3. 감지 데몬 켜기

Claude Code 채팅에서 → "install_watcher 실행해줘". 끝. 이후 무인 동작.

MCP 도구

도구설명
install_watcher감지 데몬 설치 — 한도 자동 감지+큐+예약 시작
watcher_status / uninstall_watcher데몬 상태 / 제거
scan_now지금 즉시 트랜스크립트 스캔
get_plan무엇을 언제 실행할지 — 실행 순서 + 다음 예정시각
get_status큐 + 예약 + next_run + 데몬 상태
run_task / enqueue_task지금 실행(한도면 자동 큐) / 큐에만 등록
schedule_run / cancel_schedule재실행 예약(HH:MM,+30m) / 해제
remove_task / clear_tasks / get_logs제거 / 비우기 / 로그

가벼운 CLI (선택)

데몬 없이 수동으로만 쓰고 싶으면 bash CLI:

curl -fsSL https://raw.githubusercontent.com/naryeo628/claude-token-queue/main/install.sh | bash 
ctq run "작업"      # 실행, 한도면 자동 큐+예약
ctq at  14:30       # 재실행 시각 예약
ctq status          # 큐/예약/다음 실행 예정 확인

한계

macOS launchd 전용 (Linux는 cron/systemd 백엔드 추가 필요).
감지 데몬은 고정 설치(uv tool install) 필요 — ephemeral uvx로는 KeepAlive가 깨질 수 있음.
완전 자동 실행: 리셋 시각에 큐의 명령이 무인 실행됨 — 끄려면 uninstall_watcher.
