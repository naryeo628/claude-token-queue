claude-token-queue

토큰 한도로 막힌 Claude Code 작업을 리셋 시각에 자동 재실행 · macOS
자동 감지
래퍼가 한도 에러를 직접 보고 큐에 자동 등록
정시 재실행
launchd가 지정 시각에 큐를 자동으로 비움
안전망
아직 한도면 보존 후 10분 뒤 재시도
1회성
큐 비우면 예약 자동 해제

설치

curl -fsSL https://raw.githubusercontent.com/naryeo628/claude-token-queue/main/install.sh | bash

설치 후 새 터미널을 열거나 source ~/.zshrc. 전제: claude CLI가 PATH에 있어야 함.

사용법

ctq run "LQ-7180 쿼리 최적화 검토하고 결과 알려줘"   # 실행. 한도면 자동 큐+예약
ctq add "리팩터 X 마무리"                            # 큐에만 수동 등록
ctq at  14:30                                     # 재실행 시각 예약 (+30m, +2h 가능)
ctq status                                          # 큐/예약 확인
ctq log                                             # 실행 로그
ctq drain                                           # 지금 즉시 큐 실행
ctq cancel                                          # 예약 취소
ctq clear                                           # 큐 비우기

흐름

  • ctq run "..." — 한도 걸리면 자동으로 큐 등록 + (가능하면) 리셋 시각 예약
  • 시각 자동추출 실패 시 → ctq at 14:30 직접 입력
  • 리셋 시각에 launchd가 큐를 순서대로 실행, 끝나면 예약 자동 해제

구성

명령역할
ctq runclaude -p 실행 → 한도 감지 → 큐 등록 + 시각 자동 예약
ctq add큐에 cwd|||프롬프트 추가
ctq at입력 시각으로 launchd plist 작성·로드
ctq drain예약 시각에 큐 비움 (한도 미해제면 보존+재예약)

한계

리셋 시각 자동추출은 보장 안 됨(에러 포맷 버전차) → 실패 시 ctq at로 직접 입력.
대화형 TUI 자체는 자동 캡처 불가 → 자동화하려면 ctq run으로 실행.
claude -p는 1회성 헤드리스 = 세션 컨텍스트 없음 → 프롬프트 독립적으로 작성.
